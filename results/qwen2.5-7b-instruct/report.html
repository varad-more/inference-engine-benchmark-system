Architecture Internals

vLLM — PagedAttention

  • KV cache split into fixed-size pages (blocks)
  • Block manager handles physical ↔ logical mapping
  • Prefix cache reuses blocks for repeated prefixes
  • Continuous batching with per-request token budget
  • Metrics exposed via Prometheus /metrics

SGLang — RadixAttention

  • KV cache as a Radix Tree (trie over token sequences)
  • Automatic prefix deduplication across all in-flight requests
  • Native sgl.fork() for parallel decoding branches
  • Constrained decode built-in (regex / JSON schema)
  • Metrics via /get_server_info JSON endpoint
